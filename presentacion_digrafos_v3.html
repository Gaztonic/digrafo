¿Qué representa cada vértice

G = (V, E) → donde V = tareas del sistema, E = dependencias entre ellas

⬤  Vértice (nodo) — conjunto V

Cada vértice del grafo representa una tarea del sistema. No contiene información sobre el orden: simplemente identifica una actividad que debe realizarse.

En este problema, el conjunto de vértices es:

V = { T1, T2, T3, T4, T5 }

→  Arista dirigida — conjunto E

Cada arista representa la relación de dependencia entre dos tareas: indica que una tarea debe completarse antes de que pueda comenzar la siguiente.

El conjunto de aristas de este problema es:

E = { (T1,T3), (T2,T3), (T3,T4), (T4,T5) }

Cuatro aristas = cuatro relaciones de precedencia.

T1 → T3 T1 debe completarse antes de iniciar T3
T2 → T3 T2 debe completarse antes de iniciar T3
T3 → T4 T3 debe completarse antes de iniciar T4
T4 → T5 T4 debe completarse antes de iniciar T5

Qué indica la dirección de cada arista

La dirección de una arista Ti → Tj indica que Ti debe ejecutarse antes que Tj. Es decir, establece un orden de precedencia obligatorio entre tareas.

1

Relación asimétrica

Si T1 → T3 existe, no implica que exista T3 → T1. La dependencia va en un único sentido.

2

Habilitación de inicio

La tarea destino (Tj) no puede comenzar hasta que todas sus predecesoras en el grafo hayan finalizado.

3

Grafo no dirigido = inválido

Sin dirección, el grafo no puede representar que T1 precede a T3 pero no al revés. La dirección es indispensable.

Qué permite visualizar el grafo

El dígrafo no es simplemente una ilustración: es una herramienta formal que revela la estructura completa del sistema de tareas de un vistazo.

🔗

Dependencias entre tareas

Permite ver con claridad qué tareas dependen de otras y cuáles son independientes entre sí (T1 y T2 no dependen de nadie).

📋

El orden de ejecución posible

El grafo exhibe los posibles órdenes topológicos: secuencias lineales en que las tareas pueden ejecutarse respetando todas las restricciones.

⚠️

Cuellos de botella

Vértices con muchas aristas entrantes (como T3) revelan puntos críticos donde convergen múltiples dependencias y que bloquean el avance.

🚦

Paralelismo disponible

Las tareas sin aristas entre sí (T1 y T2) pueden ejecutarse en paralelo, algo que el grafo evidencia de forma inmediata.

Estructura DAG — Propiedades y garantías

Importancia de la ausencia de ciclos
y qué garantiza esta estructura

⚠️  ¿Qué es un ciclo y por qué es un problema?

Un ciclo ocurre cuando existe una secuencia de aristas que parte de un vértice y regresa a él mismo. En el contexto de tareas, esto crea una dependencia circular imposible de resolver.

Ejemplo hipotético (INVÁLIDO):
Si existiera T3 → T4 → T3, entonces T3 necesitaría que T4 termine, pero T4 necesitaría que T3 termine. Ninguna podría comenzar jamás.

EJEMPLO INVÁLIDO — CICLO
→ Ninguna tarea puede comenzar nunca. Sistema bloqueado.

Por eso el modelo debe ser acíclico: la ausencia de ciclos es condición necesaria para que el sistema de tareas sea ejecutable en la práctica.

✅  Qué garantiza la estructura G = (V, E)

El hecho de que el modelo sea un DAG (Directed Acyclic Graph) proporciona las siguientes garantías formales:

🔢

Existencia de orden topológico

Todo DAG admite al menos un orden topológico: una secuencia lineal de vértices que respeta todas las aristas. En este caso: T1, T2, T3, T4, T5.

✅

Ejecutabilidad del sistema

Garantiza que siempre existe una manera de completar todas las tareas sin quedar bloqueado por dependencias imposibles.

🔁

Identificación de paralelismo

La estructura permite identificar qué tareas no tienen dependencias entre sí y pueden ejecutarse simultáneamente (T1 y T2 en este caso).

📐

Modelo formal y verificable

La definición G = (V, E) permite aplicar algoritmos formales (BFS, DFS, Kahn) para detectar ciclos, calcular caminos críticos y planificar la ejecución.

Conclusión

El dígrafo G = (V, E) con V = {T1, T2, T3, T4, T5} y E = {(T1,T3), (T2,T3), (T3,T4), (T4,T5)} modela el sistema de tareas de manera precisa, ejecutable y sin ambigüedad. Su naturaleza acíclica garantiza que el sistema puede resolverse. Su naturaleza dirigida garantiza que el orden de ejecución queda completamente definido.
